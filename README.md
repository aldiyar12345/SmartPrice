# SmartPrice – AI Price Navigator

Одностраничный лендинг агрегатора маркетплейсов с профилем, корзиной, избранным и единым поиском с фильтрами.

## Установка

```bash
cd smartprice
npm install
```

## Запуск в dev-режиме

```bash
npm run dev
```

Откройте URL из консоли (по умолчанию `http://localhost:5173`) — вы увидите страницу `SmartPrice` с:

- hero-блоком
- преимуществами
- статистикой с анимированными счётчиками
- единым поиском с фильтрами и категориями
- блоком «как это работает»
- отзывами
- CTA-баннером
- страницей профиля с корзиной и избранными товарами (переключается через пункт «Профиль» в навбаре).

## Документация API (Swagger)

При запущенном бэкенде перейдите по адресу:
`http://localhost:8000/api/docs/` — интерактивная документация (создана с помощью drf-spectacular).

## Запуск автотестов

Бэкенд содержит базовые тесты приложения. Для запуска перейдите в папку `backend` и выполните:
```bash
python manage.py test
```

## Продакшен-деплой

1. На сервере DigitalOcean установите `docker` и `docker-compose`.
2. Создайте на сервере папку `~/smartprice` и разместите там репозиторий.
3. В `backend/.env` настройте данные для production (см. `backend/.env.example`).
4. В Namecheap настройте DNS:
   - `A @ -> 165.22.253.31`
   - `A www -> 165.22.253.31`
5. В GitHub Actions добавьте secrets:
   - `DO_HOST` = `165.22.253.31`
   - `DO_USERNAME` = ваш SSH-пользователь на Droplet
   - `DO_SSH_KEY` = приватный SSH-ключ для подключения к серверу
6. На сервере создайте `backend/.env` на основе `backend/.env.example`.
   Убедитесь, что в нём указан `ALLOWED_HOSTS` с вашим доменом и IP.
7. Запустите на сервере:
```bash
docker-compose -f docker-compose.prod.yml up -d --build
```

## Последние обновления (CI/CD, Оптимизация, Бэкапы)

В рамках работы над архитектурой проекта и развертыванием были добавлены следующие улучшения:
- **CI/CD Pipeline (GitHub Actions)**: Автоматическое тестирование кода (Pytest) и деплой на сервер DigitalOcean при пуше в основную ветку.
- **Celery & Redis**: Настроены фоновые задачи для тяжелых процессов (обновление цен, обработка парсеров), чтобы не блокировать основной поток приложения.
- **Оптимизация БД**: Добавлены индексы (B-Tree) к таблице `Product` в PostgreSQL для ускорения фильтрации и полнотекстового поиска.
- **Автоматические бэкапы**: Настроен скрипт `scripts/setup_cron.sh` для регулярного резервного копирования базы данных с помощью cron.
- **Покрытие тестами**: Настроен `pytest` с генерацией отчетов о покрытии кода (`pytest-cov`).
- **Production-ready Docker**: Разделение конфигурации `docker-compose.yml` (для разработки) и `docker-compose.prod.yml` (для production сервера).

## Аудит безопасности (SEC-002 - SEC-007)

В рамках работы по безопасности были внесены следующие изменения, которые можно проверить:

1. **Вход через Google OAuth 2.0 (SEC-002)**: На бэкенде добавлен эндпоинт `/accounts/google-login/`. На фронтенде (в формах входа и регистрации) предусмотрена логика интеграции Google Sign-In, которая отправляет токен на проверку.
2. **RBAC — роли admin и user (SEC-003)**: Реализовано через модель `UserProfile` (чтобы не ломать существующую БД кастомной моделью юзера). Для проверки:
   - Зарегистрируйте первого пользователя — он автоматически получит роль `admin`. Все последующие — `user`.
   - Попробуйте отправить запрос на защищенный эндпоинт `/api/accounts/admin/metrics/` от имени обычного `user` — вы получите `403 Forbidden`.
3. **CAPTCHA на регистрацию и вход (SEC-004)**: Бэкенд проверяет параметр `captcha_token` во всех эндпоинтах авторизации (`/accounts/login/`, `/accounts/register/`, `/accounts/test-register/submit/`). На фронтенде предусмотрено поле для передачи этого токена. В dev-режиме (или без `RECAPTCHA_SECRET_KEY`) проверка пропускает запросы.
4. **Уязвимости OWASP (SEC-005)**: 
   - Настроено ограничение частоты запросов (Rate Limiting — защита от брутфорса, A07) для эндпоинтов авторизации — не более 10 запросов в минуту с одного IP. Вы можете проверить это, отправив несколько быстрых запросов на логин.
   - SQL инъекции (A03) предотвращаются встроенным ORM Django (сырые запросы `raw` отсутствуют).
5. **Заголовки безопасности и HTTPS (SEC-006)**: В `backend/config/settings.py` добавлены заголовки `SECURE_SSL_REDIRECT`, `X_FRAME_OPTIONS = 'DENY'`, HSTS. Они активируются, когда `DEBUG=False`.
6. **Аудит секретов (SEC-007)**: Проведена проверка на утечки через `trufflehog`. Уязвимые ключи в файле `.env` добавлены в `.gitignore`, чтобы предотвратить их попадание в репозиторий.
